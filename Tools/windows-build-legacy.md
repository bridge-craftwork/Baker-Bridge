# Windows build path — previous implementation (removed)

The Baker Bridge content pipeline was originally a Windows toolchain. It has been fully
replaced by the Mac-native pipeline (`build-mac.sh`, using the Rust `dealer3` and
`bridge-wrangler` binaries). The Windows path and its split, machine-specific artifacts
were **removed from the working tree in Phase B** (issue #21) to end the mac/windows
divergence documented in `passer-fill-bba-redesign.md`.

This file is a pointer, not a spec. **The detailed history lives in git** — use
`git log --follow` / `git show` on the paths below to recover any of it.

## What was removed (and its Mac replacement)

| Removed (Windows lineage) | Replaced by |
|---|---|
| `Fill_Hands.ps1` | `fill_hands.py` (dealer3) + `passer_reroll.py` (BBA-reject) |
| `convert_pbns_to_pdfs.ps1` | `convert_pbns_to_pdfs.py` (bridge-wrangler) |
| `Convert_Deal00_to_PDF.ps1` | `Convert_deal00_to_PDF.py` |
| `Test-FilterFolders.ps1` | `test_filter_folders.py` |
| `auction-fixes/validate_bba.py` (SSH→Windows BBA) | native `bba-cli` via `audit_passers.py` / `passer_reroll.py` |
| SSH/Windows BBA validation inside `fill_hands.py` (`--validate-bba`, `call_bba_batch`, `ssh_runner`, `BBA_CLI_PATH = C:\BBA-CLI\bba-cli`) | native `bba-cli` (`--auction-prefix`), no SSH |
| `constructed_hands-windows.csv`, `BakerBridgeFull-windows.csv` | single machine-independent `constructed_hands.csv` / `BakerBridgeFull.csv` + committed `passer_cache.csv` |
| `pbns-windows/`, `pdfs-windows/`, `Rotations-windows/` | regenerated by the Mac build into `pbns/`, `pdfs/`, `Rotations/` |

## Why

The Windows and Mac builds each ran the non-deterministic passer fill independently and
cached the results in separate `-windows` copies, so a dealer-machine file sourced from the
Windows lineage and a handout sourced from the Mac tree could carry **different passer
hands for the same board** (this bit a real class on 2026-07-14). Phase B makes the passer
fill deterministic (seeded BBA-reject + committed `passer_cache.csv`) and unifies on one
lineage, so there is a single source of truth again.

See `passer-fill-bba-redesign.md`, `passer-fill-phase-b.md`, and issue #21.
